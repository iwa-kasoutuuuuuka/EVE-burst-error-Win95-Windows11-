# EVE Burst Error (Win95版) 高画質化・Windows 11適用 実行計画書

## 目標 (Goal)
PS2版から抽出・変換した高画質・高音質アセット（MP4/MP3）と、Windows 11互換用DLL（`version.dll`）を統合し、
新規フォルダ `E:\EVE_Win95改` にてゲームが完全に動作する環境を構築します。

## ユーザーレビュー事項 (User Review Required)
> [!IMPORTANT]
> オリジナルの `EVE.EXE` は古い MCI (Media Control Interface) を使用してAVI動画やWAV音声を再生しています。
Windows 11 上でMP4/MP3をこのMCI経由で再生するよう経路をリダイレクト（偽装）しますが、
Windows側のOSコーデック状況によってはMCIでのMP4再生がうまくいかない場合があります。
> まずは最もシンプルな「MCI命令内のファイル名や拡張子のみをMP4/MP3へ置き換えて渡す」アプローチで実装を進めます。

> [!IMPORTANT]
> `E:\EVE_Win95改` にオリジナルファイルをコピーする際、
巨大なデータファイル群 (`EVE1.DAT`, `EVE2.DAT`, `EVE3.DAT` 計 約1.8GB) もすべてコピーします。
ドライブEの容量に問題がないかご確認ください。

## 実装ステップ (Proposed Changes)

### 1. ベースゲームデータのコピー
`C:\Users\AAA\.gemini\antigravity\WinEVE\EVE_Game` などに存在するオリジナルのWin95版資産を
`E:\EVE_Win95改` へコピーします。
- `EVE.EXE`, `EVE.HLP`, `EVE.CNT`, `EVE*.DAT`
- ※必要に応じてPythonスクリプト・またはコマンドで一括コピー処理を実施します。

### 2. メディアファイルとパス・マッピングの構築
- `E:\PS2解析\CONVERTED_MEDIA` 以下に生成された MP4/MP3 などの変換済みアセットを、
ゲームから参照しやすいように `E:\EVE_Win95改\MEDIA` 等の格納用フォルダへ配置（またはシンボリックリンク等で参照）します。
- 旧来の `*.AVI` や `*.WAV` といった要求ファイル名を、新しい `file_xxxx.mp4` などに
結びつけるためのマッピング定義（iniファイル・またはDLL内への埋め込み）を作成します。

### 3. `version.dll` (互換ラッパー) の拡張・ビルド
現在の大まかなパス・リダイレクト処理（`Hook_CreateFileA`）をさらに強化します。

#### [MODIFY] `version.cpp`
- **マッピング変換ロジック追加**: `MOVIE.AVI` → `MEDIA\KOJIROU\file_000184.mp4` のように、
旧ファイル名要求を実際のモダンな変換済みファイルパスへ書き換えるロジック。
- **MCI関数のフック追加**: `winmm.dll` の `mciSendStringA` と `mciSendCommandA` に対する 
IAT（インポートアドレステーブル）フックを追加実装します。これにより、
ゲーム側が `play MOVIE.AVI` という文字列を送ってきた際、ラッパー側で 
`play E:\EVE_Win95改\MEDIA\...\file_000184.mp4` と書き換えて実OSに指示を出します。

#### [NEW] ビルドと配置
- 修正後の `version.cpp` を64bit OS上の32bit向けDLL（x86）として `cl.exe` や `clang++` 等でコンパイルし、
`version.dll` を `E:\EVE_Win95改` へ配置します。

## 検証計画 (Verification Plan)

### 1. 手動での起動検証
- 上記すべてのファイルが揃った `E:\EVE_Win95改\EVE.EXE` を起動し、タイトル画面に遷移できることを確認します。
- オープニングムービーがMP4から再生されるか、クリック時などのBGMや効果音が意図通り鳴るかを確認します。

### 2. ログを用いたリダイレクト検証
- 同時出力される `eve_compat.log` を確認し、元来のCD-ROMへのアクセスや AVIの呼び出しが、
正しくローカルの MP4 / MP3 へと向かっていることをログからバリデーションします。
